# 完全還沒題目：先從這裡開始

如果你連題目都還沒想好，先不要跑 Python，也不要急著寫第 1 章。

你現在最需要的是讓 agent 問你問題，幫你把生活、工作或課堂經驗整理成可能研究方向。

## 你只要先貼這段給 agent

```text
我還沒有論文題目。請先不要幫我寫論文，也不要直接給我正式題目。

請先問我 8-10 個問題，從以下面向了解我的狀況：
1. 我的工作/生活/課堂經驗
2. 我感興趣但還不確定的關鍵字
3. 我可能拿得到的資料
4. 指導教授或系所偏好的方法
5. 我的時間限制
6. 我最害怕卡住的地方

問完後，請幫我整理 3-5 個可能研究方向。
每個方向請列出：可能研究問題、需要找的文獻關鍵字、可能資料、方法難度、最大風險、下一步。

請不要編造文獻，不要直接寫正文。
```

## 你不用先懂這些

| 詞 | 白話意思 |
| --- | --- |
| seed papers | 一開始用來認識領域的核心候選文獻，還不代表已經驗證。 |
| literature map | 把文獻分群，看目前研究做到哪裡、缺哪裡。 |
| gap radar | 幫你找可能研究缺口的表格，不是證明 gap 已存在。 |
| domain hint | 給 agent 的領域提示，例如海事安全、教育科技、服務管理。 |
| promise-delivery check | 檢查第 1 章講的概念，後面章節有沒有真的做到。 |

## 今天只做三件事

1. 讓 agent 問你問題。
2. 從回答中選 1-2 個最可能方向。
3. 對選定方向建立文獻地圖，再拿去跟老師討論。

## 什麼時候才要碰第 1 章

等你至少能說清楚：

- 我想研究哪個現象。
- 我可能拿得到什麼資料。
- 我找到哪些候選文獻關鍵字。
- 我準備問老師哪些問題。

如果這些還不清楚，先不要改 `source/shadow/ch1.md`。
