# PSC 文獻地圖示例

這是一個示範：如果學生只知道「我想做 PSC」，如何把它變成文獻地圖。這裡的 PSC 指 Port State Control，中文常譯為港口國管制或港口國監督。

> 這不是本工具的唯一主題，只是示範如何從一個模糊詞長出研究方向。

## 第一步：先確認 PSC 不是題目

`PSC` 只是關鍵字，不是研究問題。

比較好的題目方向會長得像：

- PSC 檢查缺失資料如何用於船舶風險預測？
- 台灣 PSC 實務與國際研究趨勢之間有什麼落差？
- 不同 PSC regime 的缺失分類與滯留判斷有何差異？
- AI 或資料分析方法能否改善高風險船舶篩選？

## 第二步：先抓候選研究群集

先不要把 AI 產生的分類當成事實。請把下列項目當成「待查證的候選群集」，再用 Google Scholar、Semantic Scholar、Elicit、SciSpace、Litmaps、ResearchRabbit、Connected Papers 或學校資料庫交叉搜尋：

- PSC regime 或制度績效評估
- substandard vessels 的目標篩選
- inspection outcome / detention 的決定因素
- PSC inspections 的影響
- 缺失資料與船舶風險分析
- 資料分析方法與 focus trends

## 第三步：可站的位置

| 位置 | 可能題目 | 適合的碩士方法 |
| --- | --- | --- |
| 台灣情境 gap | 國際 PSC 研究趨勢與台灣實務資料之對照 | 文件分析、專家訪談 |
| 方法 gap | PSC 缺失資料在風險分群上的應用 | 次級資料分析、分類模型 |
| 實務 gap | PSC 風險篩選模型與現場檢查決策之落差 | 個案研究、訪談 |
| 制度 gap | 不同 MOU / regime 對缺失與滯留的治理差異 | 比較研究 |

## 第四步：不要直接跳到高難度模型

如果學生是碩專班、時間有限，建議先問：

- 是否能取得台灣或公開 PSC 缺失資料？
- 是否有足夠樣本做統計或模型？
- 如果沒有資料，是否能改成訪談/文件分析？
- 指導教授接受量化、質化，還是混合方法？

## 第五步：可問 AI 的第一句

```text
我只知道想做 PSC（Port State Control）相關論文。請先不要幫我寫文獻回顧。
請幫我把 PSC 拆成 6 個可研究方向，每個方向列出：
1. 核心文獻關鍵字
2. 可能 seed papers 類型
3. 國內外 gap
4. 台灣碩士生可能取得的資料
5. 適合的研究方法
6. 題目太大或資料不可得的風險
```

## 可查證的公開起點

請先用這些搜尋語找到 seed papers，再建立自己的文獻矩陣：

| Purpose | Search query |
| --- | --- |
| Review papers | `"port state control" "literature review"` |
| Data methods | `"port state control inspections" "data analysis"` |
| Deficiency and risk | `"port state inspection deficiencies" "ship risk"` |
| Detention determinants | `"port state control" detention determinants` |
| Regime comparison | `"port state control" regime performance` |

所有 AI 產出的文獻說法都必須回到原文確認。
